# Contributing

Merci de contribuer ! Résumé rapide :
- Fork / clone le repo.
- Crée une branche feature/XXX.
- Respecte les règles de `CODING_RULES.md`.
- Ajoute des tests et des stories pour tout nouveau composant.
- Ouvre une PR en décrivant les changements et en remplissant la checklist (voir TASKS.md).

Processus PR
- Title : `[feat|fix|chore] - courte description`
- Description : Pourquoi, quoi, comment tester.
- Inclure screenshots / GIFs pour changements UI.
